# NetXMS Weekly Digest — 20 to 28 April 2026

Summary of changes merged into the `master` branch and issue tracker
activity between Monday 20 April and Tuesday 28 April 2026.

## At a Glance

- **3 commits** landed on `master`, all on 20 April, all by
  Victor Kirhenshtein. Engineering attention this period was clearly
  weighted toward triage and review rather than merge volume — there
  were no further merges between 21 and 28 April.
- **16 issues opened** (10 enhancements, 5 bugs, 1 unlabelled).
- **16 issues closed**, including several long-standing legacy items
  (NX-2008 / NX-2155 / NX-2218 / NX-2425 / NX-2081) and a
  **major security issue** in the `filemgr` subagent (#3212).

## Highlights

### Code changes merged (20 April)

- **Configurable automatic column resize for table views** — first
  the table/tree viewer infrastructure landed (`0f2673c`), then the
  remaining built-in views were converted to the configurable variant
  (`353423b`). Together these close #3185.
- **Build hygiene** — `-Wtemplate-id-cdtor` enabled where the compiler
  supports it (GCC 14+, clang) (`c802997`).

### Notable issue tracker outcomes

- **Security: filemgr subagent path-traversal** — #3212 was opened and
  closed in the same week (labels: `bug`, `major`, `agent`, `security`).
  Worth noting for downstream packagers.
- **Long-standing backlog cleared:**
  - #2217 (NX-2008) Object tools in switch port context.
  - #2290 (NX-2081) Object details from event log source via right-click.
  - #2363 (NX-2155) Expand-all child containers.
  - #2425 (NX-2218) ACL for Action Execution log and Notification log.
  - #2631 (NX-2425) Object Tools on interfaces.
- **NXSL surface widened:**
  - #3141 Manually set peer interface for `Interface` class from NXSL.
  - #3201 Methods to set node postal address (country, region, etc.).
- **Server / DB housekeeping:**
  - #3203 Drop the pre-2.1 database upgrade path from `nxdbmgr`.
  - #3204 Move `idata`/`tdata` table creation from metadata rows to code.
  - #3195 Slow-query investigation on MySQL `idata` tables.
- **Packaging / installer:**
  - #3197 Installer cannot create DB on Windows with Portuguese
    regional settings (closed same week as opened).
  - #3152 Investigate `ExecStopPost` in systemd units for automatic
    DB unlock.

> Many of the items above were closed without a corresponding `master`
> commit appearing this week — fixes are presumably staged on
> integration branches or in review. The next digest should pick up
> the merges.
